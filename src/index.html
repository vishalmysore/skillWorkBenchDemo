📚 What are "sources"?

Each source is a mock corpus — a realistic text summary of a real-world codebase or technical document (e.g. an audio processing library's API signatures, or an FFmpeg encoding manual). No real audio files, videos, or APIs are used. The demo simulates what an LLM would receive as input when generating a skill.

⚗ What is a "skill"?

A SKILL.md artifact is a reusable procedural package (from Claude Code's Agent Skills interface) that encodes how to accomplish a class of tasks. It can be stored, versioned, and reused across agents. The demo shows how different generation methods produce different quality skills from the same source material.

🔬 What does the pipeline do?

Stage 1–2: Construct a knowledge graph from the source. Stage 3: The selected generation method (e.g. SkillSeekers) distills the source into a SKILL.md. Stage 4: Static rule-based checks score 6 quality axes. Stage 5: The skill is executed against test cases — pass@3 measures if any of 3 trials succeed.

📊 What do the numbers mean?

pass@3 comes from the paper's benchmark across 187 real tasks — it's the aggregate rate, not computed live. Static scores (0–100) measure structural quality of the SKILL.md. The key finding: static quality ≠ execution success. SkillNet scores highest statically but SkillSeekers wins on pass@3.
